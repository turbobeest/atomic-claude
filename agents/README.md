## Quality Framework

All agents are audited against 10 dimensions:

1. **Identity Clarity** - Clear persona and interpretive lens
2. **Knowledge Grounding** - Authoritative sources (official docs, RFCs, academic)
3. **Instruction Density** - 8-12 Always, 4-6 Mode-specific, 3-5 Never rules
4. **Vocabulary Calibration** - 15-20 domain-specific terms
5. **Tool Integration** - Proper audit/solution mode support
6. **Error Handling** - Graceful degradation patterns
7. **Context Efficiency** - Token-optimized definitions
8. **Consistency** - Follows tier templates
9. **Completeness** - All required sections present
10. **Actionability** - Clear, executable guidance

Each agent includes embedded audit metadata in YAML frontmatter.

## Contributing

After improving agents:
```bash
git add -A
git commit -m "improve: <agent-name> - <brief description>"
git push
```

Projects using this agent library will receive updates via `cd agents && git pull`.
